# AI와 협업하여 인터랙티브 프로토타입 만들기
## SettingsPopup 프로젝트 사례

**🔗 완성된 프로토타입:** https://sunyoungkim3.github.io/svpro-settings/

---

## 1. 프로젝트 개요

기획자로서 새로운 기능의 컨셉을 전달할 때 PPT나 구두 설명만으로는 한계가 있습니다. 특히 인터랙티브한 UI/UX를 설명할 때는 실제 동작하는 프로토타입이 훨씬 효과적입니다. 이 문서는 **Claude Sonnet 4.5** (Visual Studio의 GitHub Copilot을 통해 사용)를 활용하여 **Settings Popup** 프로토타입을 개발한 실제 사례를 통해, 비개발자도 AI와 협업하여 동작하는 프로토타입을 만드는 방법을 제시합니다.

### 프로젝트 결과물
- **기술 스택**: React + Vite
- **배포 방식**: GitHub Pages (자동 배포)
- **개발 기간**: 몇 시간 이내
- **핵심 기능**: 
  - 다중 탭 구조의 설정 팝업
  - 실시간 검색 및 필터링
  - 반응형 디자인
  - 인터랙티브한 UI 요소들

## 2. SettingsPopup 프로젝트 실제 프롬프트 워크플로우

### Phase 1: 프로젝트 초기 설정

**핵심은 컴포넌트와 기능 구현이지 프로젝트 세팅이 아닙니다.**

이 프로젝트에서는 프로젝트 초기 설정 단계를 스킵했습니다. 이전에 사용하던 React + Vite 초기 설정이 계승되었고, 원하는 특정 설정(포트 번호, base path 등)이 있다면 먼저 선언하고 사용하면 됩니다.


**프로토타입 제작 시 권장 사항:**
- 기존 프로젝트 구조가 있다면 그대로 활용
- 새로 시작한다면 최소한의 설정만 요청
- 빠른 프로토타입이 목적이므로 복잡한 초기 설정보다는 **기능 구현에 집중**

### Phase 2: 레이아웃 구조 정의

**첫 번째 단계는 전체 레이아웃을 명확히 정의하는 것입니다.**

**화면 레이아웃 구조:**
```
┌─────────────────────────────────────────────────┐
│  Settings                                       │
├──────────┬──────────────────────────────────────┤
│  General │                                      │
│  Display │        설정 콘텐츠 영역                │
│  Export  │                                      │
│  Print   │                                      │
└──────────┴──────────────────────────────────────┘
```

```
프롬프트: "Settings Popup 프로토타입을 만들어줘. 
          - 팝업 형태로 화면 중앙에 표시 (1200px × 800px)
          - 좌측에 세로 탭 메뉴 (General, Display Setting, Export, Print, Assay Pack)
          - 우측에 각 탭의 설정 내용이 표시되는 영역
          - 우측 상단에 닫기 버튼
          - 하단에 저장 버튼"
```

**핵심 포인트:**
- UI 배치를 구체적으로 설명 (위치, 크기)
- 탭 구조와 전체 흐름을 먼저 확립
- 세부 내용은 다음 단계에서 추가

### Phase 3: 탭별 설정 메뉴 구성

**각 탭에 필요한 설정 항목과 UI 형식을 명시합니다.**

```
프롬프트: "General 탭에 다음 설정 항목들을 추가해줘:
          1. Language - 드롭다운
          2. PCR Raw Data Type - 드롭다운 (필수 항목)
          3. Data Loading Methods - 다중 선택 버튼 (Manual, Preset, LIMS, CSV, Barcode)
          4. Sample ID - 라디오 버튼 (PLRN, PLTN)
          5. Target Abbreviation - 토글 스위치
          6. 폴더 경로 설정 - 텍스트 입력 필드 + 찾아보기 버튼
             - Open PCR Data File Folder Path
             - Open LIMS File Folder Path
             - Export Folder Path
             - Save Folder Path"
```

**작성 요령:**
- 설정 항목 이름을 정확히 명시
- UI 형식을 명확히 지정 (드롭다운, 라디오버튼, 토글, 텍스트 입력 등)
- 필수 항목이나 특별한 조건이 있으면 함께 표시
- 다른 탭(Display Setting, Export, Print)도 같은 방식으로 작성

### Phase 4: 설정 메뉴 인터랙션 추가

**기본 페이지 구성이 완료되면 각 설정 메뉴의 동작 규칙과 인터랙션을 추가합니다.**

```
프롬프트: "Data Loading Methods 다중 선택 버튼에 다음 규칙을 적용해줘:
          - LIMS와 CSV는 동시 선택 불가 (하나를 선택하면 다른 하나 자동 해제)
          - Barcode는 단독 선택 불가 (다른 옵션 중 최소 1개 선택 필요)
          - 규칙 위반 시 경고 메시지 표시
          - 정보 아이콘(ℹ️) 추가하고 호버 시 규칙 설명 툴팁 표시"

프롬프트: "저장되지 않은 변경사항 경고 기능을 추가해줘.
          - 설정 값 변경 시 '변경됨' 상태로 표시
          - 탭 전환 시 변경사항이 있으면 경고 팝업 표시
          - 경고 내용: '저장되지 않은 변경사항이 있습니다. 계속하시겠습니까?'
          - 계속하기/취소 버튼 제공"

프롬프트: "특정 설정 변경 시 분석 탭에 영향을 준다는 추가 경고를 표시해줘.
          영향을 주는 설정: PCR Raw Data Type, Sample ID, Result View, 
          C(t) Value Digit, Sample Index Setting, Target on/off"
```

**인터랙션 작성 요령:**
- 하나의 설정 메뉴에 대한 규칙을 한 번에 하나씩 추가
- 조건과 결과를 명확히 설명
- 사용자 피드백 방식 지정 (메시지, 툴팁, 색상 변경 등)

### Phase 5: 복잡한 기능 구현 (기능 위주 작성)

**단순 설정 항목이 아닌 복잡한 기능은 기능 위주로 상세하게 작성합니다.**

**예시: Display Setting > Target on/off 탭**

```
프롬프트: "Display Setting 탭 아래에 Target on/off 서브탭을 추가해줘.
          
          1. Assay 검색 기능
          - 상단에 검색창 배치
          - 검색어 입력 시 Assay 이름으로 실시간 필터링
          
          2. Assay 테이블 표시
          - 테이블 헤더: Name, Intended Use, Target 개수, Pack 정보
          - Assay 목록을 테이블 형태로 나열
          
          3. Assay 클릭 시 Target 목록 노출
          - Assay 행을 클릭하면 해당 Assay의 Target 목록이 확장되어 표시
          - 다시 클릭하면 축소
          
          4. Target 테이블 표시
          - 각 Target의 이름과 현재 상태 표시
          - 각 Target마다 on/off 토글 스위치 추가
          - 토글 변경 시 분석 탭 영향 경고 표시"
```

**복잡한 기능 작성 요령:**
- 기능을 단위별로 나누어 설명 (검색, 테이블, 클릭 동작, 토글 등)
- 각 기능의 동작 방식을 구체적으로 명시
- 데이터 표시 형식 (테이블 헤더, 내용 등) 명확히 지정
- 사용자 액션과 결과를 명확히 연결

### Phase 6: 배포 준비

**프로토타입을 웹에서 공유할 수 있도록 준비합니다.**

```
프롬프트 시퀀스:
1. "배포할 수 있도록 빌드 파일을 만들어줘"
2. "변경한 내용을 Git에 저장하고 업로드해줘"
3. "GitHub에서 자동으로 배포되도록 설정 파일을 만들어줘"
4. "배포 후 접속할 수 있는 웹 주소를 알려줘"
```

**배포 완료 후:**
- 생성된 웹 주소(예: https://사용자명.github.io/프로젝트명/)를 통해 누구나 접속 가능
- 코드를 수정하고 Git에 업로드하면 자동으로 웹사이트에 반영됨
- 팀원들과 링크를 공유하여 실제로 동작하는 프로토타입 시연 가능

## 3. 기획자를 위한 실전 체크리스트

**프로토타입 시작 전:**
- [ ] 필요한 화면과 기능 목록 작성
- [ ] 간단한 스케치나 박스 그림 정도의 레이아웃 준비

**개발 진행 중:**
- [ ] 각 단계마다 실행하여 결과 확인
- [ ] 원하는 결과와 다르면 즉시 수정 요청

**완성 후:**
- [ ] 다양한 시나리오로 테스트
- [ ] Git에 커밋 및 푸시
- [ ] 배포하여 링크 공유

이 접근법을 통해 스크린샷으로는 설명하기 복잡한 설정 기능을, 실제로 작동하는 프로토타입으로 효과적으로 커뮤니케이션할 수 있습니다.
